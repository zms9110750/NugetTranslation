# 命令树：
```
NugetTranslation <code> [flags]
- translate pack<string[]>
- cache
  - build pake<string[]>
  - forget pack-name<string[]>
```
## 翻译
解析pack，对其foreach，对每个包执行操作。

### 确定包

创建一个字典<string：包名，HashSet<包版本>：需要处理的包版本>

对包字符串做以下解析

- packname：视作packname@*处理。
- packname@ver
  -  pack:包名
  -  ver：包版本
      -  单个版本，例如`*`，`*-*`，`1.3.*`，`1.0`，使用nugetapi的最佳匹配确定一个版本。加入到处理版本。
      -  范围版本，例如[1,2)
          - 如果左右任意有预览版，使用nugetapi，对所有匹配版本加入到处理。
          - 如果左右都没有预览版，匹配版本后，剔除预览版，只加入正式版到处理。
          - nugetapi可以检查是不是预览版。
          - nugetapi，可以检查浮动范围有没有最大值来确定是不是范围版本。
  - 如果不是这个格式，报错。     
 

### 下载包

遍历字典，再遍历里面的HashSet。此过程使用并发：

1. 从nuget服务器上获取资源，得到压缩包。
2. 检查本地是是否有对应的xml文件并写入
    1. 流打开压缩包内容
    2. 用sha256检查其与本地的值。
    3. 如果一样，就什么也不做
    4. 如果不一样，或本地没有这个文件，执行写入。
3. 保存这个过程得到的所有zip，暂时不进一步解开内容。这些zip全部驻留内存

### 预检

1. 遍历下载包的zip资源。传递这个过程的包对应的名字与路径：
2. 检查里面所有lib，ref文件夹，.xml结尾的文件。
3. 解析为Xdoc。对里面的成员进行遍历。
4. 从缓存确认存在性。
    1. 使用TryGetAsync方法。需要有bool返回值的方法或返回一个带bool属性的对象。
    2. 如果缓存不存在，加入一个HashSet字典。Value为Xelement的ToString
5. 对所有xmlDoc遍历完成后，统计需要处理的set。
    1. 用Microsoft.ML.Tokenizers，统计value，确认有多少M的token。
    2. 打出日志，{name}包{ver}有{count}条需要翻译，预计{tokenCount/1e6:F3}M个token。

### 翻译

1. 获取agent，为他创建消息。
#### agent初始消息：
```
system:[
你是一个翻译器。目标语言代码：{{_language}}。
你仅翻译 XML、JSON 等配置文件中的文本内容（如 summary、param、returns 等标签内的文字），
不翻译格式标签、属性名、属性值中的标识符。
你必须调用工具，把你的翻译内容作为参数。
工具会验证和使用你的翻译格式。你不需要输出对话内容。
]
```
#### 包的readme，如果有的话。
两个消息不使用字符串拼接，使用一个对话里的多个对话组成。把readme模拟为上传文件。

```
use:[为了让你了解包的背景，这是他的readme，readme]
```
#### 这一个成员的字符串
使用一个对话里的多个对话组成。把xml模拟为上传文件。
```
use:[这是你要翻译的内容, xml]
```

2. 为要翻译的xml成员，创造一个对象。这个对象应该有
```
{
xelement：原始只读xml
xelement：Parse以后的xml，公开读，私有写

string? ParseXml(string xml)
{
  Xelement.Parse(xml)
如果解析不正确，捕获错误，返回错误内容为字符串。
如果解析正确，验证name的内容和原始只读的name一样。
如果不一样，返回错误内容的字符串。

如果通过验证，把翻译后结果的那个成员，赋值为这个解析结果。
然后字符串返回null。
}
}
```

对每一个翻译，都要单独构建一个agent，并把这个对象的实例方法作为委托，给agent当作工具调用。让agent通过这个工具来进行翻译。

3. 中间件截断：注册中间件。如果检查到工具调用返回null，截断对话，终止。翻译已经完成。
4. 写入缓存。key为翻译前xml的ToString，value为翻译后xml的ToString。并且需要tag，tag内容为xml的name。

### 汇报
翻译工作并发完成。并且使用原子操作：
1.  收集每次翻译所用的token消耗
2.  每执行100条翻译，汇报进度。`当前完成{已翻译}/{总数}，消耗token:命中{}，输入{}：输出{}`
他们的单位都是M，都是精确到小数点后三位。
3.  对每个错误都要汇报。例如name对不上，xml无法解析。

### 写文件
对doc进行替换。对每个成员，从缓存中，解析为xml，保留格式。替换到doc中。然后输出为文件夹。
按理来说这里应该有所有的缓存，都找得到。如果这里有缓存缺失，直接报错。


## 遗忘缓存
格式：packname@name

效果，由于输入缓存时，对他们都构建了tag。
因此不需要知道翻译的前后全文。
对name执行tag删除。
但是，进行检查，如果是全部由空白，或通配符（*）组成的字符串，跳过这一个执行。

## 构建缓存
从本地的这个包，收集所有的版本和目标框架。
对翻译前后，如同上述翻译过程的逻辑，进行写缓存。
